# 部署架构与指导

## 一、架构概览

本项目为前后端分离的全栈应用，支持以下两种部署模式：

| 模式 | 适用场景 | 复杂度 |
|------|----------|--------|
| 单机部署（同进程） | 个人使用、演示 | 低 |
| 分离部署（独立服务） | 生产环境、团队使用 | 中 |

---

## 二、环境要求

| 依赖 | 最低版本 | 说明 |
|------|----------|------|
| Node.js | 18.x+ | 需支持 ES Modules（`"type": "module"`） |
| npm | 8.x+ | 包管理器 |
| 操作系统 | Linux / macOS / Windows | 无特殊限制 |
| 内存 | 512MB+ | 取决于上传文件大小 |
| 网络 | 可访问 LLM API | 需能连接 Anthropic 或兼容接口 |

---

## 三、环境变量配置

复制 `.env.example` 为 `.env`，填写以下变量：

```bash
cp .env.example .env
```

```env
# LLM API 密钥（必填）
ANTHROPIC_AUTH_TOKEN=your_api_key_here

# API 接入点（可选，默认使用 Anthropic 官方）
# 官方：https://api.anthropic.com
# 阿里云百炼：https://coding.dashscope.aliyuncs.com/apps/anthropic
ANTHROPIC_BASE_URL=https://api.anthropic.com

# 使用的模型（可选，默认 claude-3-5-sonnet-20241022）
ANTHROPIC_MODEL=claude-3-5-sonnet-20241022

# 后端服务端口（可选，默认 3001）
PORT=3001
```

### 支持的 LLM 接入点

| 服务商 | BASE_URL | 推荐模型 |
|--------|----------|----------|
| Anthropic 官方 | `https://api.anthropic.com` | `claude-3-5-sonnet-20241022` |
| 阿里云百炼 | `https://coding.dashscope.aliyuncs.com/apps/anthropic` | `glm-5` |
| 自建代理 | 自定义 URL | 取决于代理配置 |

## 四、本地开发部署

### 1. 安装依赖

```bash
npm install
```

### 2. 启动开发环境（前后端同时）

```bash
npm run dev:all
```

这会并行启动：
- 前端 Vite 开发服务器：`http://localhost:5173`
- 后端 Express 服务器：`http://localhost:3001`

前端通过 Vite 代理将 `/api/*` 请求转发到后端，无需手动配置跨域。

### 3. 单独启动

```bash
# 仅启动前端
npm run dev

# 仅启动后端（支持热重载）
npm run dev:server
```

## 五、生产环境部署

### 方案 A：前后端分离部署（推荐）

```
┌─────────────────────────────────────────────────────────────┐
│                        用户浏览器                             │
└──────────────┬──────────────────────────┬───────────────────┘
               │ 静态资源请求               │ /api/* 请求
               ▼                          ▼
┌──────────────────────┐    ┌─────────────────────────────────┐
│   静态文件服务器        │    │        Node.js 后端服务          │
│  (Nginx / CDN)        │    │     (Express，端口 3001)         │
│                       │    │                                 │
│  dist/                │    │  POST /api/upload               │
│  ├── index.html       │    │  POST /api/query (SSE)          │
│  ├── assets/          │    │  GET  /api/session/:id          │
│  └── ...              │    │                                 │
└──────────────────────┘    └─────────────────────────────────┘
```

#### 步骤 1：构建前端

```bash
npm run build
# 输出到 dist/ 目录
```

#### 步骤 2：配置 Nginx（静态文件 + API 反向代理）

```nginx
server {
    listen 80;
    server_name your-domain.com;

    # 前端静态文件
    root /path/to/ts-excel-reader/dist;
    index index.html;

    # SPA 路由支持
    location / {
        try_files $uri $uri/ /index.html;
    }

    # API 反向代理到后端
    location /api/ {
        proxy_pass http://localhost:3001;
        proxy_http_version 1.1;
        proxy_set_header Connection '';          # SSE 需要
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_buffering off;                     # SSE 需要禁用缓冲
        proxy_cache off;                         # SSE 需要禁用缓存
        proxy_read_timeout 300s;                 # SSE 长连接超时
    }
}
```

#### 步骤 3：启动后端服务

```bash
# 直接启动
npm run server

# 或使用 PM2 管理进程（推荐）
npm install -g pm2
pm2 start "npm run server" --name ts-excel-reader
pm2 save
pm2 startup
```

### 方案 B：Docker 部署

#### Dockerfile（后端）

```dockerfile
FROM node:20-alpine

WORKDIR /app

COPY package*.json ./
RUN npm ci --only=production

COPY server/ ./server/
COPY tsconfig*.json ./

EXPOSE 3001

CMD ["node", "--env-file=.env", "--import", "tsx/esm", "server/index.ts"]
```

#### docker-compose.yml（前后端一体）

```yaml
version: '3.8'

services:
  backend:
    build: .
    ports:
      - "3001:3001"
    environment:
      - ANTHROPIC_AUTH_TOKEN=${ANTHROPIC_AUTH_TOKEN}
      - ANTHROPIC_BASE_URL=${ANTHROPIC_BASE_URL}
      - ANTHROPIC_MODEL=${ANTHROPIC_MODEL}
      - PORT=3001
    restart: unless-stopped

  frontend:
    image: nginx:alpine
    ports:
      - "80:80"
    volumes:
      - ./dist:/usr/share/nginx/html
      - ./nginx.conf:/etc/nginx/conf.d/default.conf
    depends_on:
      - backend
    restart: unless-stopped
```

```bash
# 构建前端
npm run build

# 启动所有服务
docker-compose up -d
```

### 方案 C：云平台部署

#### Vercel（仅前端）+ Railway/Render（后端）

**后端部署到 Railway：**
1. 连接 GitHub 仓库
2. 设置环境变量（ANTHROPIC_AUTH_TOKEN 等）
3. 启动命令：`npm run server`
4. 获取后端 URL（如 `https://your-app.railway.app`）

**前端部署到 Vercel：**
1. 连接 GitHub 仓库
2. 构建命令：`npm run build`
3. 输出目录：`dist`
4. 添加环境变量或 `vercel.json` 配置 API 代理：

```json
{
  "rewrites": [
    {
      "source": "/api/:path*",
      "destination": "https://your-app.railway.app/api/:path*"
    }
  ]
}
```

## 六、SSE 部署注意事项

本项目使用 **Server-Sent Events（SSE）** 实现流式响应，部署时需特别注意：

| 配置项 | 要求 | 说明 |
|--------|------|------|
| 代理缓冲 | 必须关闭 | Nginx: `proxy_buffering off` |
| 响应缓存 | 必须关闭 | Nginx: `proxy_cache off` |
| 连接超时 | 建议 300s+ | 复杂查询可能需要较长时间 |
| HTTP 版本 | HTTP/1.1 | SSE 不支持 HTTP/2 多路复用 |
| CDN | 谨慎使用 | 部分 CDN 会缓冲 SSE 响应 |

## 七、性能与限制

| 限制项 | 当前值 | 说明 |
|--------|--------|------|
| 文件大小 | 50MB | Multer 配置，可调整 |
| 查询结果行数 | 最多 100 行 | query_data 工具限制 |
| Agent 迭代次数 | 最多 10 轮 | 防止无限循环 |
| 会话持久性 | 内存存储 | 服务重启后丢失 |
| 并发会话 | 无限制（受内存约束） | 大文件多会话时注意内存 |

### 内存估算

```
单个会话内存占用 ≈ Excel 文件大小 × 3~5 倍（JS 对象开销）
例：10MB Excel → 约 30~50MB 内存
建议：生产环境配置 1GB+ 内存，并定期重启清理
```

## 九、故障排查

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 前端无法连接后端 | 端口未开放 / CORS 配置 | 检查 PORT 环境变量，确认防火墙规则 |
| SSE 流式响应中断 | 代理缓冲未关闭 | Nginx 添加 `proxy_buffering off` |
| LLM API 调用失败 | API Key 无效 / 网络不通 | 检查 ANTHROPIC_AUTH_TOKEN 和 BASE_URL |
| Excel 解析失败 | 文件格式损坏 | 尝试重新保存 Excel 文件 |
| 内存不足 | 大文件 + 多会话 | 增加服务器内存或限制并发 |
| 服务重启后会话丢失 | 内存存储特性 | 属于已知限制，需重新上传文件 |
