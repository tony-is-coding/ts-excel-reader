# 整体项目目录说明

## 项目概述

**ts-excel-reader** 是一个基于 TypeScript 的全栈应用，允许用户上传 Excel 文件后通过自然语言与数据对话。后端集成 Claude AI（兼容 Anthropic API 格式）作为智能 Agent，前端提供实时流式交互界面。

---

## 完整目录结构

## 核心文件说明

### 前端核心文件

| 文件 | 职责 | 关键特性 |
|------|------|----------|
| `src/App.tsx` | 根布局组件 | 侧边栏拖拽调整宽度（180-480px）、上传/聊天视图切换 |
| `src/components/ChatUI.tsx` | 聊天界面 | SSE 流式渲染、Agent 步骤可视化、Markdown 渲染 |
| `src/components/FileUpload.tsx` | 文件上传 | 拖拽支持、50MB 限制、.xlsx/.xls/.csv 格式 |
| `src/components/Sidebar.tsx` | 侧边栏 | Sheet 切换、列类型徽章（STR/NUM/DATE/BOOL/MIX） |
| `src/components/ResultTable.tsx` | 结果表格 | 点击列头排序、数字本地化格式（zh-CN）、最大高度 400px |
| `src/components/CodeBlock.tsx` | 代码展示 | Python 语法高亮、一键复制、可折叠 |
| `src/hooks/useSession.ts` | 状态管理 | 会话生命周期、文件上传、SSE 事件解析 |
| `src/types/index.ts` | 类型定义 | 全局接口：ExcelFile、Session、AgentStep、Message 等 |

### 后端核心文件

| 文件 | 职责 | 关键特性 |
|------|------|----------|
| `server/index.ts` | 服务器主文件 | Express 路由、Multer 上传、Agent 循环、工具执行 |
| `server/tsconfig.json` | TS 配置 | 服务端独立编译配置，target ES2022 |

### 配置文件

| 文件 | 职责 |
|------|------|
| `vite.config.ts` | Vite 构建配置，开发时代理 `/api` 到 3001 端口 |
| `tsconfig.json` | 前端 TypeScript 严格模式配置 |
| `tsconfig.node.json` | Vite 构建工具的 TypeScript 配置 |
| `package.json` | 依赖管理与 npm 脚本 |
| `.env.example` | 环境变量模板（ANTHROPIC_AUTH_TOKEN、BASE_URL、MODEL、PORT） |

## npm 脚本说明

```bash
npm run dev          # 启动前端开发服务器（Vite，端口 5173）
npm run dev:server   # 启动后端开发服务器（tsx watch，端口 3001）
npm run dev:all      # 同时启动前后端（concurrently）
npm run build        # 生产构建（tsc + vite build）
npm run preview      # 预览生产构建
npm run server       # 生产模式启动后端
```

## 数据流向说明

```
test_datas/          ← 测试数据（手动上传到前端）
     ↓
FileUpload.tsx       ← 用户选择文件
     ↓
POST /api/upload     ← 发送到后端
     ↓
server/index.ts      ← XLSX 解析，存入内存 sessions Map
     ↓
ChatUI.tsx           ← 用户输入自然语言问题
     ↓
POST /api/query      ← SSE 流式请求
     ↓
Agent Loop           ← Claude 调用工具查询数据
     ↓
SSE Events           ← 实时推送到前端
     ↓
ResultTable.tsx      ← 展示查询结果
```
