# 核心业务流程图

## 1. 文件上传流程

```
用户操作                    前端（React）                   后端（Express）
   │                           │                               │
   ├─ 拖拽/点击选择文件 ────────►│                               │
   │                           ├─ 校验文件类型                  │
   │                           │  (.xlsx/.xls/.csv)            │
   │                           ├─ 校验文件大小（≤50MB）          │
   │                           │                               │
   │                           ├─ POST /api/upload ───────────►│
   │                           │  FormData { file }            ├─ Multer 接收文件
   │                           │                               ├─ 校验 MIME 类型
   │                           │                               ├─ XLSX.read() 解析
   │                           │                               ├─ 遍历所有 Sheet
   │                           │                               ├─ 推断列类型
   │                           │                               │  (string/number/
   │                           │                               │   date/boolean/mixed)
   │                           │                               ├─ 生成 sessionId (uuid)
   │                           │                               ├─ 存入内存 sessions Map
   │                           │                               │
   │                           │◄─────────────────────────────┤
   │                           │  { sessionId, file, sheets }  │
   │                           │                               │
   │                           ├─ 更新 session 状态             │
   │                           ├─ 渲染 Sidebar（Schema）        │
   │◄──────────────────────────┤                               │
   │  显示文件信息 + 聊天界面     │                               │
```

## 3. SSE 事件流详解

```
时间轴 ──────────────────────────────────────────────────────────►

  连接建立
     │
     ├──► data: {"type":"agent_step","step":{"type":"thinking","status":"start"}}
     │
     ├──► data: {"type":"text_delta","content":"让我先查看数据结构..."}
     ├──► data: {"type":"text_delta","content":"我需要了解"}
     ├──► data: {"type":"text_delta","content":"Sheet 的列信息"}
     │
     ├──► data: {"type":"agent_response","response":{"tool_calls":[{"name":"get_sheet_info",...}]}}
     │
     ├──► data: {"type":"tool_result","result":{"columns":["姓名","年龄"],"sample_data":[...]}}
     │
     ├──► data: {"type":"agent_step","step":{"type":"llm_response","status":"start"}}
     │
     ├──► data: {"type":"text_delta","content":"根据数据，共有 100 行记录..."}
     │
     ├──► data: {"type":"agent_response","response":{"tool_calls":[{"name":"query_data",...}]}}
     │
     ├──► data: {"type":"tool_result","result":{"rows":[...],"total_matched":25}}
     │
     ├──► data: {"type":"final_result","answer":"找到25条符合条件的记录","result":{"columns":[...],"rows":[...]}}
     │
  连接关闭
```

## 4. 工具执行逻辑流程

### get_sheet_info

```
输入: { sheet_name, sample_rows=5 }
  │
  ├─ 从 sessions Map 获取 sheet 数据
  ├─ 获取列名列表
  ├─ 截取前 N 行样本数据
  └─ 返回: { row_count, columns, sample_data }
```

### query_data

```
输入: { sheet_name, filters[], sort_by, sort_order, limit }
  │
  ├─ 获取全量行数据
  ├─ 应用过滤器（逐行）
  │   ├─ eq / ne    → 严格相等/不等
  │   ├─ gt / lt    → 数值大小比较
  │   ├─ gte / lte  → 数值大小等于比较
  │   └─ contains   → 字符串包含（toLowerCase）
  ├─ 排序（sort_by + sort_order: asc/desc）
  ├─ 截取前 limit 行（最大 100）
  └─ 返回: { total_matched, returned, columns, rows }
```

### aggregate_data

```
输入: { sheet_name, operation, column, group_by_column, filters[] }
  │
  ├─ 获取全量行数据
  ├─ 应用过滤器（同 query_data）
  ├─ 执行聚合操作
  │   ├─ sum     → 数值求和
  │   ├─ avg     → 数值平均值
  │   ├─ min     → 最小值
  │   ├─ max     → 最大值
  │   ├─ count   → 行数统计
  │   └─ group_by → 按列分组，每组统计 count/sum/avg
  └─ 返回: 聚合结果对象
```

## 5. 错误处理流程

```
错误发生点                    处理方式                      用户感知
─────────────────────────────────────────────────────────────────
文件类型不符          前端拦截，不发送请求          提示框：不支持的文件格式
文件超过 50MB         前端拦截，不发送请求          提示框：文件过大
sessionId 无效        后端返回 400                  错误消息显示在聊天框
Excel 解析失败        后端返回 500                  错误消息显示在聊天框
LLM API 调用失败      SSE 发送 error 事件           聊天框显示错误详情
工具执行异常          返回 error 对象给 LLM         LLM 根据错误调整策略
超过 10 轮迭代        强制结束循环                  返回当前最佳结果
网络中断              SSE 连接断开                  前端状态重置为 idle
```

## 6. 数据类型推断流程

```
Excel 单元格值
      │
      ├─ 是 Date 对象？ ──────────────────────► type: "date"
      │
      ├─ 是 number？ ──────────────────────────► type: "number"
      │
      ├─ 是 boolean？ ─────────────────────────► type: "boolean"
      │
      ├─ 是 string？
      │   ├─ 可解析为数字？ ──────────────────► type: "number"
      │   ├─ 可解析为日期？ ──────────────────► type: "date"
      │   └─ 其他 ─────────────────────────────► type: "string"
      │
      └─ 同一列存在多种类型？ ─────────────────► type: "mixed"
```
