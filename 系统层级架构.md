## 各层职责详述

### 表现层（Presentation Layer）

负责用户界面渲染与交互，完全运行在浏览器中。

| 组件 | 职责 | 技术特点 |
|------|------|----------|
| `App.tsx` | 整体布局、视图切换 | 侧边栏拖拽（180-480px）、响应式布局 |
| `FileUpload.tsx` | 文件选择与上传 | 拖拽 API、文件类型/大小校验 |
| `Sidebar.tsx` | 数据 Schema 展示 | 列类型推断、Sheet 切换 |
| `ChatUI.tsx` | 对话交互主界面 | SSE 实时渲染、Agent 步骤折叠展示 |
| `ResultTable.tsx` | 结构化结果展示 | 内存排序、数字格式化（zh-CN） |
| `CodeBlock.tsx` | 代码展示 | 正则语法高亮、剪贴板 API |

### 状态层（State Layer）

单一 Hook 管理全部应用状态，避免 prop drilling。

```typescript
// 核心状态结构
interface Session {
  id: string
  file: ExcelFile | null
  sheets: SheetInfo[]
  messages: Message[]
  status: 'idle' | 'uploading' | 'querying' | 'error'
}
```

### 接口层（API Layer）

Express 路由层，处理 HTTP 请求、文件解析、SSE 连接管理。

- 文件上传使用 Multer 内存存储（不落盘）
- 查询接口使用 SSE 保持长连接，实时推送事件
- CORS 配置允许前端跨域访问

### 业务层（Business Layer）

核心 Agent 循环逻辑，协调 LLM 与数据工具：

```
迭代 N（最多10次）:
  1. 组装消息历史 + 工具定义 → 发送给 LLM
  2. 流式接收 LLM 响应
  3. 若有工具调用 → 执行 executeTool()
  4. 将工具结果追加到消息历史
  5. 若 stop_reason === 'end_turn' → 结束循环
```

### 数据层（Data Layer）

纯内存存储，适合单用户/演示场景：

- 会话数据存储在 Node.js 进程内存中
- 服务重启后所有会话丢失
- 无持久化、无数据库依赖
- XLSX 库负责将二进制文件转换为 JS 对象

### 外部服务层（External Services）

通过环境变量配置 LLM 接入点，支持：
- Anthropic 官方 API
- 阿里云百炼（DashScope）兼容接口
- 任何兼容 Anthropic API 格式的代理服务

## 关键技术决策

| 决策 | 选择 | 原因 |
|------|------|------|
| 状态管理 | 自定义 Hook（无 Redux/Zustand） | 状态简单，无需全局共享 |
| 实时通信 | SSE（Server-Sent Events） | 单向流式推送，比 WebSocket 更轻量 |
| 数据存储 | 内存 Map | 无需持久化，简化部署 |
| 样式方案 | 内联 CSS-in-JS + CSS 变量 | 无外部 CSS 框架依赖，主题切换灵活 |
| 后端语言 | TypeScript（tsx 直接运行） | 与前端共享类型，开发体验一致 |
| 文件解析 | XLSX 库（内存模式） | 支持 xlsx/xls/csv，无需临时文件 |
