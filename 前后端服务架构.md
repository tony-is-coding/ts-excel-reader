## 前端架构

### 运行环境
- 开发服务器：Vite Dev Server（默认端口 **5173**）
- 生产构建：静态 SPA，输出至 `dist/`
- API 代理：开发时 `/api/*` → `http://localhost:3001`

### 组件树

```
App.tsx                        ← 根组件，布局管理
├── FileUpload.tsx             ← 文件上传（拖拽 / 点击）
├── Sidebar.tsx                ← 文件元数据 & Schema 浏览器
│   ├── 文件信息（名称、大小、Sheet 数）
│   ├── Sheet 选择器
│   └── 列类型徽章（STR / NUM / DATE / BOOL / MIX）
└── ChatUI.tsx                 ← 主聊天界面
    ├── 消息列表（user / assistant / system）
    ├── Agent 步骤可视化
    │   ├── thinking（实时流式文本）
    │   ├── llm_response（LLM 输出 + 工具调用）
    │   └── tool_result（工具执行结果）
    ├── ResultTable.tsx        ← 可排序数据结果表格
    └── CodeBlock.tsx          ← 语法高亮代码块
```

### 状态管理

使用自定义 Hook `useSession.ts` 集中管理所有状态，无第三方状态库：

```
useSession
├── session: Session           ← 当前会话（文件、Sheet、消息列表）
├── uploadFile(file)           ← POST /api/upload
├── sendMessage(question)      ← POST /api/query（SSE 流式）
│   ├── 解析 text_delta        ← 实时文本增量
│   ├── 解析 agent_step        ← Agent 推理步骤
│   ├── 解析 agent_response    ← LLM 完整响应
│   ├── 解析 tool_result       ← 工具执行结果
│   ├── 解析 final_result      ← 最终答案
│   └── 解析 error             ← 错误事件
└── clearSession()             ← 清除会话
```

### 数据类型体系（types/index.ts）

```typescript
ExcelFile      → 文件元数据（id, name, size, sheets, uploadedAt）
SheetInfo      → Sheet 信息（name, columns, rowCount）
ColumnInfo     → 列信息（name, type, sampleValues）
AgentStep      → Agent 步骤（id, type, iteration, status, timing, content）
Message        → 消息（id, role, content, timestamp, code, result, steps）
QueryResult    → 查询结果（columns, rows, rowCount, truncated）
Session        → 会话（id, file, sheets, messages, status）
```

## 后端架构

### 运行环境
- 运行时：Node.js（ES Modules）
- 端口：**3001**（由 `PORT` 环境变量控制）
- 启动方式：`tsx` 直接执行 TypeScript

### 中间件栈

```
Express App
├── cors()                     ← 跨域支持
├── express.json()             ← JSON 请求体解析
└── multer({ storage: memoryStorage, limits: 50MB })  ← 文件上传
```

### API 端点

| 方法 | 路径 | 功能 |
|------|------|------|
| POST | `/api/upload` | 上传 Excel 文件，返回 sessionId 和 Schema |
| POST | `/api/query` | 自然语言查询，SSE 流式返回 Agent 推理过程 |
| GET | `/api/session/:id` | 获取会话元数据 |

### 会话存储

```typescript
// 纯内存存储，服务重启后丢失
const sessions = new Map<string, SessionData>();

interface SessionData {
  fileName: string;
  sheets: Map<string, Record<string, unknown>[]>;  // sheet名 → 行数据
  columns: Map<string, string[]>;                   // sheet名 → 列名列表
}
```

### Agent 工具定义

后端向 Claude 暴露 3 个结构化工具：

```
get_sheet_info
├── 输入：sheet_name, sample_rows（默认5）
└── 输出：row_count, columns, sample_data

query_data
├── 输入：sheet_name, filters[], sort_by, sort_order, limit（最大100）
├── filters：{ column, operator(eq|ne|gt|lt|gte|lte|contains), value }
└── 输出：total_matched, returned, columns, rows

aggregate_data
├── 输入：sheet_name, operation(sum|avg|count|min|max|group_by), column, group_by_column, filters[]
└── 输出：聚合结果（含 count/sum/avg 等）
```

### Agent 循环逻辑

```
POST /api/query
│
├── 1. 验证 sessionId 和 question
├── 2. 设置 SSE 响应头
├── 3. 构建数据摘要（文件名、Sheet、列名、样本行）
│
└── Agent Loop（最多 10 轮）
    ├── 调用 Claude API（流式）
    ├── 发送 text_delta 事件（实时文本）
    ├── 发送 agent_step 事件（推理步骤）
    ├── 发送 agent_response 事件（完整 LLM 输出）
    ├── 执行工具调用 → executeTool()
    ├── 发送 tool_result 事件
    ├── 若 stop_reason === 'end_turn' → 退出循环
    └── 发送 final_result 事件
```

## 前后端通信协议

### 文件上传

```
POST /api/upload
Content-Type: multipart/form-data

Request:  FormData { file: File }
Response: {
  sessionId: string,
  file: ExcelFile,
  sheets: SheetInfo[]
}
```

### 流式查询（SSE）

```
POST /api/query
Content-Type: application/json

Request:  { sessionId: string, question: string }
Response: text/event-stream

事件类型：
  data: {"type":"text_delta","content":"..."}
  data: {"type":"agent_step","step":{...}}
  data: {"type":"agent_response","response":{...}}
  data: {"type":"tool_result","result":{...}}
  data: {"type":"final_result","answer":"...","result":{...}}
  data: {"type":"error","message":"..."}
```

## AI 集成

### LLM 配置

通过环境变量配置，兼容 Anthropic API 格式：

```
ANTHROPIC_AUTH_TOKEN   ← API 密钥
ANTHROPIC_BASE_URL     ← 接口地址（支持阿里云百炼等兼容端点）
ANTHROPIC_MODEL        ← 模型名称（如 glm-5、claude-3-5-sonnet 等）
```

### 数据上下文注入

每次查询时，后端将以下信息注入 Claude 的系统提示：
- 文件名
- 所有 Sheet 名称
- 每个 Sheet 的列名和类型
- 每个 Sheet 的前 3 行样本数据
